# IntClaw Plugin 架构文档

## 项目概述

IntClaw Connector Plugin 是一个基于 OpenClaw SDK 的企业内部机器人连接器插件，使用 Stream 模式连接，无需公网 IP，支持 WebSocket 流式响应。

## 核心模块

### 1. 连接管理 (Connection Manager)

**职责：**
- 管理 WebSocket 连接生命周期
- 实现应用层心跳检测
- 处理连接重连逻辑
- 消息去重处理

**特性：**
```
┌─────────────────────────────────────────────────────────────┐
│                    Connection Manager                       │
├─────────────────────────────────────────────────────────────┤
│  心跳配置                                                    │
│  • 心跳间隔: 10 秒                                           │
│  • 超时阈值: 20 秒                                           │
│  • 重连策略: 指数退避 (1s → 30s)                             │
├─────────────────────────────────────────────────────────────┤
│  消息处理                                                    │
│  • 立即确认回调 (防止重发)                                   │
│  • 消息去重 (5 分钟 TTL)                                     │
│  • 三阶段日志 (接收/解析/处理)                                │
├─────────────────────────────────────────────────────────────┤
│  连接状态                                                    │
│  • 建立 → 认证 → 活跃 → 断线 → 重连                          │
│  • 长消息处理保护 (防止心跳超时)                              │
└─────────────────────────────────────────────────────────────┘
```

### 4. 流式响应架构

```
┌─────────────────────────────────────────────────────────────────────┐
│                      Streaming Response Flow                        │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  OpenClaw Agent                                                     │
│       │                                                             │
│       │ 1. onReplyStart                                             │
│       ├──────────────────► 发送 response.in_progress                │
│       │                   (初始化流式会话)                           │
│       │                                                             │
│       │ 2. onPartialReply / deliver (kind="block")                  │
│       ├──────────────────► 发送 response.output_text.delta          │
│       │                   (增量文本更新)                             │
│       │                   ↓                                         │
│       │              可重复多次...                                   │
│       │                                                             │
│       │ 3. deliver (kind="final")                                   │
│       ├──────────────────► 发送 response.completed                  │
│       │                   (完成响应，最终文本)                       │
│       │                                                             │
└───────┴─────────────────────────────────────────────────────────────┘
                             │
                             ▼
                    ┌─────────────────┐
                    │ WebSocket Adapter│
                    │ (ws-out-adapter) │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │  IntClaw Server │
                    │   WebSocket     │
                    └─────────────────┘
```

## 功能特性矩阵

| 功能类别 | 功能项 | 支持状态 | 说明 |
|---------|--------|---------|------|
| **连接** | WebSocket 连接 | ✅ | Stream 模式，无需公网 IP |
| | 心跳检测 | ✅ | 10秒间隔，20秒超时 |
| | 自动重连 | ✅ | 指数退避策略 |
| | 多账号 | ✅ | 支持配置多个账号 |
| **消息接收** | 文本消息 | ✅ | 支持 text 类型 |
| | 富文本 | ✅ | 支持 markdown/richText |
| | 图片 | ✅ | 自动下载和处理 |
| | 文件 | ✅ | 支持各种文件类型 |
| | 音频 | ✅ | 支持 amr/mp3 格式 |
| | 视频 | ✅ | 支持 mp4 等格式 |
| | Word/PDF | ✅ | 自动提取文本 |
| **消息发送** | 文本/Markdown | ✅ | 自动格式检测 |
| | 媒体文件 | ✅ | 图片/视频/音频/文件 |
| | 主动推送 | ✅ | 单聊/群聊 |
| | 流式响应 | ✅ | 实时增量更新 |
| **安全** | 白名单 | ✅ | DM 白名单控制 |
| | 群聊策略 | ✅ | open/allowlist 模式 |
| | 认证 | ✅ | Token 自动刷新 |

## 配置示例

```json
{
  "channels": {
    "intclaw-connector": {
      "enabled": true,
      "clientId": "your-app-key",
      "clientSecret": "your-app-secret",
      "streaming": true,
      "debug": false,
      "groupPolicy": "allowlist",
      "groupAllowFrom": ["user:allowed-user-id"],
      "systemPrompt": "你是一个企业助手机器人"
    }
  }
}
```

## API 端点

```
IntClaw API (主动消息)
├── POST /v1.0/robot/oToMessages/batchSend     (单聊消息)
├── POST /v1.0/robot/groupMessages/send        (群聊消息)
└── POST /v1.0/robot/interactiveCards/send     (AI Card)

IntClaw OAPI (媒体上传)
├── POST /v1.0/media/upload                    (通用上传)
├── POST /v1.0/media/images/upload             (图片上传)
└── GET /v1.0/media/files/download             (文件下载)

WebSocket Stream (实时连接)
└── wss://[endpoint]/stream                    (Stream 模式)
```

## 错误处理

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Error Handling                               │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  连接错误                                                           │
│  • 400 Bad Request     → clientId/clientSecret 格式错误              │
│  • 401 Unauthorized    → 认证失败，凭据无效                          │
│  • 连接超时            → 自动重连，指数退避                          │
│                                                                     │
│  消息错误                                                           │
│  • 解析失败            → 记录日志，跳过处理                          │
│  • 重复消息            → 检测 messageId，自动忽略                    │
│  • 发送失败            → 兜底错误消息，用户通知                      │
│                                                                     │
│  媒体错误                                                           │
│  • 上传失败            → 降级为文本提示                              │
│  • 文件过大            → 按类型限制检查                              │
│  • 格式不支持          → 友好错误提示                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
