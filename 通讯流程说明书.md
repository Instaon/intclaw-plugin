# 通讯流程说明书

## 架构概览

```
远程服务端 ←(WebSocket 双向)→ 渠道插件 ←(进程内函数调用)→ 内置 SDK
```

- 远程服务端 ↔ 插件：**WebSocket 双向长连接**
- 插件 ↔ SDK：**同进程内函数调用，回调模式**

---

## 消息生命周期

```
远程服务端 ──ws推送──→ 插件（解析消息）
                              │
                        调用 SDK 派发方法，传入消息
                              │
                        SDK 启动内置 AI 模型
                              │
                        AI 模型以流式（streaming）方式逐段产出文本
                              │
                        SDK 通过回调将文本传回插件
                              │
                        插件在回调中通过 WebSocket 发送回复
                              │
远程服务端 ←──ws回复──────────────┘
```

### 关键点

整个过程本质就是**方法调用**：
1. 插件调用 SDK 的方法，把消息传进去
2. SDK 内部启动 AI 模型拿到回复
3. SDK 调用插件预先注册的回调函数，把回复文本传回来

没有网络通讯，没有中间件，就是同进程内的函数调用。
