## 消息协议规格

### 认证流程
1. Client 连接时在 Header 中发送 API Key
2. Client 发送 `auth_request` 消息
3. Server 返回 `auth_response` 消息

### 消息类型
- `auth_request`: 认证请求
- `auth_response`: 认证响应
- `incoming_message`: 服务端 → OpenClaw 的消息
- `outgoing_message`: OpenClaw → 服务端的消息
- `ping`/`pong`: 心跳检测

### 配置示例
```json5
{
  channels: {
    intclaw: {
      enabled: true,
      wsUrl: "wss://api.intclaw.example.com/ws",
      apiKey: "your-api-key-here",
      dmPolicy: "pairing",
      groupPolicy: "allowlist",
    },
  },
}
```
